# Адаптация RTCW для современных Linux с использованием SDL2 и x64

## Обзор изменений

Этот документ описывает процесс адаптации исходного кода Return to Castle Wolfenstein (RTCW) для сборки на современных Linux-системах с использованием:
- **CMake** в качестве системы сборки
- **SDL2** для доступа к окну, управлению, звуку и OpenGL
- **x64** архитектуры с правильным обращением с указателями

## Ключевые проблемы оригинального кода

### 1. Устаревшая система сборки
Оригинальный код использует устаревшую систему сборки на основе `pcons` и Makefile, которые не поддерживают современные стандарты.

### 2. Прямой доступ к X11 и OSS
Оригинальный код напрямую использует:
- X11 для оконного режима (`linux_glimp.c`)
- OSS (`/dev/dsp`) для звука (`linux_snd.c`)
- Прямой ввод через терминал и X11 события

### 3. Проблемы с x64 архитектурой
В коде多处 используются приведения типов между указателями и `int`, что вызывает проблемы на 64-битных системах:
```c
// Проблематично для x64:
static int asmCallPtr = (int)doAsmCall;  // Усечение указателя!

// Правильно для x64:
static uintptr_t asmCallPtr = (uintptr_t)doAsmCall;
```

## Реализованные изменения

### Структура файлов SDL2

Создана новая директория `src/sdl2/` со следующими файлами:

#### 1. `sdl_glimp.c` - OpenGL рендеринг через SDL2
**Заменяет:** `src/unix/linux_glimp.c`

**Ключевые функции:**
- `GLimp_Init()` - Инициализация OpenGL контекста через SDL2
- `GLimp_SetMode()` - Создание окна с нужным разрешением
- `GLimp_EndFrame()` - SwapBuffers через SDL_GL_SwapWindow
- `GLimp_SetGamma()` - Установка гаммы через SDL_SetWindowGammaRamp

**Особенности x64:**
- Использует `intptr_t` и `uintptr_t` для всех приведений указателей
- Корректная работа с 64-битными адресами памяти

#### 2. `sdl_input.c` - Ввод через SDL2
**Заменяет:** `src/unix/linux_joystick.c` и часть `unix_main.c`

**Ключевые функции:**
- `IN_Init()` - Инициализация клавиатуры, мыши, джойстиков
- `IN_Frame()` - Обработка событий SDL2
- `IN_MapKey()` - Преобразование SDL2 scancode в коды RTCW
- `IN_MouseEvent()` - Обработка движений и кнопок мыши

**Особенности x64:**
- Правильное хранение состояний клавиш в массивах с корректным размером
- Использование `SDL_GetRelativeMouseState()` для точного трекинга

#### 3. `sdl_snd.c` - Звук через SDL2 Audio
**Заменяет:** `src/unix/linux_snd.c`

**Ключевые функции:**
- `SNDDMA_Init()` - Инициализация аудиоустройства SDL2
- `SDL_AudioCallback()` - Callback для заполнения буфера
- `SNDDMA_Shutdown()` - Завершение работы аудио

**Особенности x64:**
- Корректное выделение памяти для аудио буферов
- Использование `uint64_t` для временных меток

#### 4. `sdl_main.c` - Точка входа и системные функции
**Заменяет:** части `src/unix/unix_main.c`

**Ключевые функции:**
- `main()` - Точка входа приложения
- `Sys_Init()` - Инициализация SDL2 подсистем
- `Sys_Quit()` - Завершение работы
- `Sys_Milliseconds()` - Получение времени через `SDL_GetTicks64()`
- `Sys_Error()` - Обработка ошибок с SDL MessageBox

**Особенности x64:**
- `SDL_GetTicks64()` вместо 32-битного времени
- Корректные приведения при работе с указателями функций

#### 5. `sdl_shared.c` - Общие утилиты
**Заменяет:** части `src/unix/unix_shared.c`

**Ключевые функции:**
- `Sys_GetClipboardData()` / `Sys_SetClipboardData()` - Буфер обмена
- `Sys_Mkdir()` / `Sys_unlink()` - Работа с файловой системой
- `Sys_FindFirst()` / `Sys_FindNext()` - Поиск файлов
- `Sys_OpenURL()` - Открытие URL в браузере

#### 6. `sdl_local.h` - Заголовочный файл
Объявления всех функций и структур для SDL2 платформы.

### CMakeLists.txt

Создан современный `CMakeLists.txt` с:

1. **Настройка стандартов:**
   - C11 для C кода
   - Позиционно-независимый код (PIC) для x64
   - Предупреждения компилятора

2. **Поиск зависимостей:**
   ```cmake
   find_package(SDL2 REQUIRED)
   find_package(OpenGL REQUIRED)
   ```

3. **Определение архитектуры:**
   ```cmake
   if(CMAKE_SIZEOF_VOID_P EQUAL 8)
       add_definitions(-DARCH_X64)
   endif()
   ```

4. **Целевые определения:**
   - `rtcw_client` - основной исполняемый файл
   - `game`, `cgame`, `ui` - игровые DLL

## Критические исправления для x64

### 1. Приведение указателей

**До (проблематично):**
```c
static int asmCallPtr = (int)AsmCall;
void *ptr = (int)some_pointer;
```

**После (корректно):**
```c
static uintptr_t asmCallPtr = (uintptr_t)AsmCall;
void *ptr = (void *)(intptr_t)some_value;
```

### 2. Вычисление смещений структур

**До:**
```c
#define FOFS(x) ((int)&(((gentity_t *)0)->x))
```

**После:**
```c
#define FOFS(x) ((intptr_t)&(((gentity_t *)0)->x))
```

### 3. Функции для работы с памятью

Использование `intptr_t` и `uintptr_t` из `<stdint.h>` для всех операций, связанных с адресами памяти.

## Сборка проекта

### Требования

```bash
# Ubuntu/Debian
sudo apt-get install cmake libsdl2-dev libgl1-mesa-dev

# Fedora/RHEL
sudo dnf install cmake SDL2-devel mesa-libGL-devel
```

### Процесс сборки

```bash
cd /workspace
mkdir build && cd build
cmake ..
make -j$(nproc)
```

### Запуск

```bash
./rtcw_client +set fs_game main
```

## Совместимость

### Поддерживаемые платформы
- Linux x86_64 (основная цель)
- Linux i686 (с ограничениями)

### Требуемые версии
- CMake >= 3.16
- SDL2 >= 2.0.8
- OpenGL >= 1.5 (совместимость)
- GCC >= 7 или Clang >= 5

## Известные ограничения

1. **VM-код (виртуальная машина)**
   - Файлы `vm_x86.c` требуют дополнительной адаптации для x64
   - Рекомендуется использовать интерпретируемый VM (`vm_interpreted.c`)

2. **Ассемблерные вставки**
   - Оригинальный код содержит x86 ассемблер
   - Для x64 требуется переписать или отключить

3. **Сетевой код**
   - Некоторые части сетевого кода используют устаревшие API
   - Требуется дополнительная проверка
